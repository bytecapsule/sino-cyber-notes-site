一个实现了Manus的核心工作流程的 Claude Code Skill： Planning with Files

github.com/OthmanAdi/planning-with-files

这个技能实现了Manus的核心工作流程模式：

“Markdown是我磁盘上的‘工作记忆’。由于我以迭代方式处理信息，并且我的活跃上下文有限，Markdown文件充当了笔记、进度检查点和最终交付物的构建模块。” — Manus AI

问题

Claude Code面临以下问题：

1. 不稳定的记忆 —— TodoWrite工具在上下文重置时消失

2. 目标漂移 —— 超过50次工具调用后，原始目标被遗忘

3. 隐性错误 —— 错误没有被跟踪，因此同样的错误会重复发生

4. 上下文塞满 —— 一切都塞进上下文，而不是存储起来

解决方案：三文件模式

对于每个复杂任务，创建三个文件：

task_plan.md → 跟踪阶段和进展

notes.md → 存储研究和发现

[deliverable].md → 最终成果

循环流程

创建 task_plan.md，定义目标和阶段

进行研究 → 保存到 notes.md → 更新 task_plan.md

阅读 notes.md → 创建交付物 → 更新 task_plan.md

提交最终成果

通过在每个决策前阅读 task_plan.md，目标始终保持在关注窗口中。这就是 Manus 如何在不迷失轨迹的情况下处理约50个工具调用的方式
