稍微来唱个反调：大厂现在需要做的是 Cowork，而不是 OpenClaw

我觉得中西方大厂对 OpenClaw 的反应是截然不同且有趣的。

先说老美的大厂，Anthropic 虽然对 OpenClaw 各种围追堵截，但是在 Claude Code 上引入了很多 OpenClaw 的特性，远程、定时任务之类的。

OpenAI 是直接引入了 OpenClaw 创始人 Peter，让他负责个人 Agent 产品。

谷歌、Meta，好像没啥反应。

倒是微软，今天刚刚推出了 Copilot Cowork，一个打包了微软全家桶的面向白领的 Cowork 类 Agent 产品。

基本上，大家都还是在自己原有路线上做 Agent，激进点、敏捷点的如 Claude Code，也就是学一下 OpenClaw，除了 OpenAI 在 Agent 时代表现不太好，快速收下 Peter 之外，其他的都没有啥太应激的反应。

但国内这边画风完全不一样了。

从过年期间的各种 claw 产品发布就不停歇，kimi、minimax、网易有道、阿里通义、傅盛，到这几天的火山引擎、腾讯云、腾讯电脑管家，没发布的还有一堆，几乎所有厂商都卷进来了。

厂均一个 Claw，有的厂还不止一个。为龙虾提供服务、支持的，那就更多了。

从外界看，Claw 快成为大厂 agent 产品唯一的方向了。

但这真的对吗？

经过这一个周末的发展，OpenClaw 火出天际。但正是这个火出天际，让整个事情的信噪比开始急剧降低。

原来关心龙虾的人，是上进的、想要提效的；现在关注的，是想找点乐子的。

另外一边，从用户的角度，大厂一窝蜂的推出龙虾产品，也很难判断谁好谁不好。

市场的摩擦力急剧攀升，甚至可能会进入到休克。

我下午说，OpenClaw 在经过这一周末的全民狂欢之后，已经从一个中性偏正面的词，转向了一个中性偏负面的词。正是这个意思。

也许，从现在开始，大厂的产品带虾就不再是一个好的信号，就好像去年DeepSeek 爆火之后，所有厂商都在接，连搞模型的都在自己产品上接入，只有豆包、kimi 等少数产品一直坚持没接。

有一些厂商就是适合去做 cowork，给更多白领使用，真正改变大家的工作习惯。

如果因为龙虾的爆火，强行虾化，又虾的不伦不类，乱了自己的产品路线，还是挺可惜的。
